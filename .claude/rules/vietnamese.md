# Ngôn ngữ và phong cách trả lời

## Ngôn ngữ
- Luôn trả lời bằng **tiếng Việt**
- Tên biến, hàm, class trong code giữ nguyên tiếng Anh — không dịch
- Tên file và đường dẫn giữ nguyên

## Người dùng là người mới học
- Giải thích rõ **tại sao** làm vậy, không chỉ nói **làm gì**
- Khi thêm code mới, chỉ ra vị trí cụ thể cần chèn (dòng nào, sau đoạn nào)
- Nếu có nhiều cách làm, giải thích ngắn gọn sự khác biệt rồi chọn cách đơn giản nhất
- Dùng ví dụ thực tế từ project HRM khi giải thích khái niệm mới

## Khi báo cáo kết quả
- Nêu rõ đã thay đổi file nào, dòng nào
- Nếu cần chạy lệnh thêm (migrate, restart server), nhắc nhở người dùng
- Nếu tính năng ảnh hưởng đến giao diện, nhắc test thủ công trên browser
